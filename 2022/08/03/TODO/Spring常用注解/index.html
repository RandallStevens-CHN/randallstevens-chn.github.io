资源读取

Environment

Spring会将读取到的配置文件封装到到Environment对象当中,当需要获取属性值是可以从中获取

各种PropertySource（属性源）

@PropertySource （添加配置源）

​ 将配置文件添加到 Environment 中,与@Configuration注解配合，对其中的bean进行属性填充绑定

​ 支持 .properties ; .xml ; .yml( 需yaml解析器) 格式配置文件.

@ConfigurationProperties 、@Value （ （属性绑定）

​ 作用：属性绑定 将配置源中属性与配置类中绑定

​ @ConfigurationProperties+ @Bean 将属性绑定到bean （可以自动绑定）

​ @ConfigurationProperties单独使用即声明为配置属性（但需注册到容器中才能使用）

​ 属性项较少用@Value 多项属性时@ConfigurationProperties

​ 底层用的是Getter / Setter 进行绑定的

@EnableConfigurationProperties ( DataSourceProperties.class )

​ 将DataSourceProperties组件作为配置属性注册到容器环境

​

属性绑定需要的属性源可以是 外部引入的配置文件 或 在容器中的配置属性源类

​

依赖注入

@Autowired @Qualifier 先按照type 后按照name 若仍找不到通过@Qualifier 指定

@Resource( Class<?> type) @Qualifier 先按照name后按照 type 若仍找不到通过@Qualifier 指定

@Inject @Named 不加@Named注解，需要配置与变量名一致即可

后两个是 java官方实现的

依赖注入的三种形式

Bean作用域

​ 后三种在web应用中才有。

类型 描述
singleton IOC容器中仅有一个bean实例.默认值。。
prototype 每次都会返回一个新的对象。后续需自己管理其生命周期。
request 每次 http 请求都会产生新的 bean，该 bean 仅在当前 http request域有效。
session 每次 http 请求都会产生新的 bean，该 bean 仅在当前 http session域有效。
global session 类似标准的 http session 作用域，仅在基于 portlet 的 web 应用中有意义。
